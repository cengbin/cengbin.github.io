# 建立前端规范

开发者需要建立和遵守的规范，大致可以划分成这几个方向：

1. 开发流程规范 
2. 项目目录与文件结构规范 
3. 开发代码规范
    1. 代码质量规范
    2. 代码风格规范
4. git commit规范 
5. 前后端接口规范

统一规范的最根本目的是为了保证团队成员的一致性，从而减少沟通成本，提高开发效率。

学会热爱标准，但要确保它们不是一成不变的。

如果制定的规范不适合您的团队，请确保可以适应和重写所需要的任何规则。

它并不是要强制执行一种工作方式，而是为了帮助促进团队之间的互动 👏👏👏。
